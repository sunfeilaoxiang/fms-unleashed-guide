Introduction - À Propos d'Unleashed

▼

Unleashed est un système de gestion des stocks (Inventory Management System) utilisé par Feel My Skin pour:

  • Suivre les niveaux de stock (Track inventory levels) dans 7 entrepôts français et roumains
  • Gérer les commandes fournisseurs (Manage purchase orders) auprès de nos suppliers
  • Traiter les commandes clients (Process customer orders) pour la vente B2C et B2B
  • Enregistrer les mouvements (Record stock movements) : transferts, ajustements, assemblages
  • Générer des rapports (Generate reports) pour l'analyse et la planification
✓
Conseil: Unleashed est accessible via https://live.unleashedsoftware.com. Utilisez vos identifiants FMS pour vous connecter.

Gestion des Produits (Product Management)

▼

Les produits dans Unleashed incluent tous les articles vendus par Feel My Skin : sérums, crèmes, sets, etc.

Étape 1: Accéder à la gestion des produits

Depuis le menu principal, cliquez sur Products (Produits). Vous verrez la liste complète de tous les produits actifs.

Liste des produits / Products list

Étape 2: Consulter les détails d'un produit

Cliquez sur le nom du produit pour voir ses détails : code SKU, prix d'achat, prix de vente, description, images, fournisseurs.

Détails d'un produit / Product details
⚠
Attention: Ne modifiez le SKU ou le code produit que si vous êtes sûr. Cela peut affecter les commandes existantes.

Étape 3: Ajouter un nouveau produit

Cliquez sur + New Product. Remplissez les champs obligatoires : nom, SKU, prix d'achat, prix de vente, fournisseur par défaut.

Informations importantes pour chaque produit

  • SKU (Stock Keeping Unit): Code unique (ex: SERUM-001)
  • Barcode: Code à barres pour scans rapides
  • Unit of Measure: Unité (pièce, boîte, carton)
  • Cost Price: Prix d'achat auprès du fournisseur
  • Retail Price: Prix de vente client
  • Supplier: Fournisseur par défaut
  • Reorder Level: Stock minimum avant réapprovisionnement

Commandes Fournisseurs (Purchase Orders)

▼

Supplier orders (POs) allow us to order stock from our suppliers.

Step 1: Access Supplier Orders

Menu → Purchase Orders pour voir toutes les PO. Vous pouvez filtrer par statut : Open, Placed, Received, Completed.

Liste des commandes fournisseurs / Purchase orders list

Step 2: Create a New PO

Click on + New Purchase Order:

  1. Select the Supplier
  2. Select the destination Warehouse
  3. Add product lines
  4. Enter quantities and prices
  5. Check the total
  6. Cliquez Save
Création d'une commande fournisseur / Create purchase order

Step 3: Place the Order

Once saved, click Place Order to send the PO to the supplier. The status changes to "Placed".

Step 4: Receive Stock

When the order arrives, click Receive. Unleashed then creates a receipt to record incoming stock.

✓
Conseil: Always verify received quantities against ordered quantities. If there is a discrepancy, note it in the receipt.

Commandes Clients (Sales Orders)

▼

Sales orders (SOs) record sales to B2C and B2B customers.

Step 1: Access Sales Orders

Menu → Sales Orders pour voir toutes les SO. Filtrez par statut : Open, Completed, Backordered.

Liste des commandes clients / Sales orders list

Step 2: Create a New SO

Click on + New Sales Order:

  1. Select the Customer
  2. Select the Warehouse for shipping
  3. Ajoutez les lignes de produits
  4. Check stock availability
  5. Set the desired delivery date
  6. Cliquez Save
Création d'une commande client / Create sales order

Step 3: Prepare and Ship the Order

When the order is ready, click Dispatch or Ship. Unleashed records the stock departure and generates a shipping document.

⚠
Important: Always verify that stock is available before creating a SO. If stock is insufficient, the order will go to "Backordered".

Ajustements de Stock (Stock Adjustments)

▼

Les ajustements permettent de corriger les erreurs de comptage, enregistrer les casses, ou les pertes de stock.

Étape 1: Accéder aux ajustements

Menu → Stock → Stock Adjustments pour voir tous les ajustements.

Ajustements de stock / Stock adjustments

Étape 2: Créer un nouvel ajustement

Cliquez + New Adjustment:

  1. Sélectionnez l'Entrepôt (Warehouse)
  2. Sélectionnez le Produit (Product)
  3. Entrez la quantité ajustée (positive = ajout, négatif = retrait)
  4. Entrez un motif (Reason) : Damage, Shrinkage, Inventory Count, etc.
  5. Cliquez Save
⚠
Attention: Les ajustements de stock affectent directement les niveaux et les rapports. Enregistrez toujours un motif clair pour traçabilité.

Motifs courants d'ajustement

  • Inventory Count: Discordance trouvée lors d'un inventaire physique
  • Damage: Produits endommagés découverts
  • Shrinkage: Perte non expliquée / Pertes évaluées
  • Stock Transfer: Transfert manuel entre entrepôts
  • Sample: Prélèvement pour démonstration/test
  • Return: Retour client non enregistré en tant que tel

Transferts entre Entrepôts (Warehouse Transfers)

▼

Les transferts permettent de déplacer du stock d'un entrepôt à un autre (ex: du dépôt vers une boutique).

Étape 1: Accéder aux transferts

Menu → Stock → Warehouse Transfers.

Transferts entre entrepôts / Warehouse transfers

Étape 2: Créer un nouveau transfert

Cliquez + New Transfer:

  1. Sélectionnez l'entrepôt source (Source Warehouse)
  2. Sélectionnez l'entrepôt destination (Destination Warehouse)
  3. Ajoutez les lignes de produits
  4. Entrez les quantités à transférer
  5. Cliquez Save
Création d'un transfert / Create transfer

Étape 3: Confirmer la réception

Une fois le stock physiquement transféré et reçu, cliquez Receive pour finaliser le transfert. Le stock est alors débité de l'entrepôt source et crédité à la destination.

✓
Conseil: Planifiez les transferts régulièrement pour équilibrer le stock entre DEPOT_FEEL, les trois boutiques Paris, et les entrepôts roumains.

Vérifications Quotidiennes (Daily Checks)

▼

Chaque jour, vous devriez effectuer certaines vérifications pour maintenir la santé du système Unleashed.

Matin: Vérifier les commandes en attente (Morning: Check pending orders)

Ouvrez le Dashboard et recherchez:

  • Commandes clients en statut "Open" prêtes à être expédiées
  • Commandes fournisseurs en attente de réception
  • Stocks faibles (Stock on-hand proche du Reorder Level)
Tableau de bord quotidien / Daily dashboard view

Midi/Après-midi: Confirmer les réceptions (Midday/Afternoon: Confirm receipts)

Vérifiez les Purchase Orders en statut "Placed". Si les marchandises sont arrivées, cliquez Receive et enregistrez.

Fin de journée: Clôturer les commandes (End of day: Close orders)

Marquez les commandes expédiées comme Completed. Cela met à jour les statistiques et nettoie la liste des tâches en cours.

Vérifications de fin de journée / End of day checks

Hebdomadaire: Rapport de stock (Weekly: Stock report)

Chaque vendredi, générez un Stock Report pour voir le niveau de chaque produit par entrepôt. Identifiez les stocks faibles ou excédentaires.

Points de contrôle clés

  • ✓ Aucune commande en statut "Backorder" depuis plus de 3 jours
  • ✓ Stock minimum (Reorder Level) atteint → Créer une PO
  • ✓ Tous les transferts entre entrepôts confirmés (Received)
  • ✓ Aucune discordance entre Unleashed et le comptage physique
  • ✓ Prix de vente et d'achat à jour dans la fiche produit

Règles Critiques (Critical Rules)

▼

Ces règles doivent toujours être suivies pour éviter des erreurs coûteuses.

🔴
Règle 1: Vérifier le stock avant de vendre
Avant de créer une SO (Sales Order), assurez-vous que le stock est disponible dans le bon entrepôt. Un stock non disponible crée des backlogs.
🔴
Règle 2: Ne jamais modifier un SKU produit
Les SKU doivent rester constants. Les modifier affecte tous les historiques. Si une correction est nécessaire, créez un nouveau produit.
🔴
Règle 3: Enregistrer les motifs pour tous les ajustements
Chaque ajustement de stock (Damage, Shrinkage, etc.) doit avoir un motif documenté. Cela assure la traçabilité.
🔴
Règle 4: Confirmer les réceptions rapidement
Dès que le stock fournisseur est reçu, cliquez "Receive" dans la PO. Cela actualise les niveaux de stock en temps réel.
🔴
Règle 5: Utiliser les bons entrepôts
Utilisez toujours le bon code d'entrepôt (CLICHY, GAITE, CHATLET, DEPOT_FEEL, etc.). Une mauvaise sélection crée des discordances.
🔴
Règle 6: Ne jamais supprimer une commande
Si une commande est erronée, marquez-la "Cancelled" ou créez une note. Ne la supprimez jamais, car cela perd l'historique.
🔴
Règle 7: Vérifier les prix régulièrement
Les prix d'achat et de vente doivent être à jour. Des prix obsolètes mènent à des marges incorrectes.
🔴
Règle 8: En cas de doute, contactez un manager
Si quelque chose semble anormal (stock négatif, prix incohérent, transfert bloqué), n'essayez pas de corriger seul. Signalez-le immédiatement.
Résumé des règles critiques / Summary of critical rules

En cas de problème

Problème (Problem) Solution (Solution)
Stock négatif (-1, -5, etc.) Créer un ajustement (Adjustment) pour corriger. Vérifier l'historique des mouvements.
Commande bloquée / Cannot save Vérifier que tous les champs obligatoires sont remplis. Contacter support si l'erreur persiste.
Transfert non reçu / Stuck in transit Aller à la section Warehouse Transfers, trouver le transfert, cliquer "Receive" pour le finaliser.
Stock différent d'un inventaire physique Créer un ajustement avec motif "Inventory Count" et enregistrer la différence.
Fournisseur introuvable Vérifier l'orthographe. Si absent, créer un nouveau fournisseur via Products → Suppliers.
