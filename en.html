    Welcome to the Feel My Skin Unleashed Inventory Management Guide. This guide is your complete reference for managing inventory, customers, suppliers, orders, and reporting within Unleashed.

    What is Unleashed?

    Unleashed is a cloud-based inventory management system that tracks stock across multiple warehouses, manages sales and purchase orders, and provides detailed reporting and analytics. For Feel My Skin, it integrates with Shopify (e-commerce), POS systems (retail stores), and accounting systems.

    Feel My Skin's Current Setup

    • Warehouses: DEPOT_FEEL (main), CLICHY (store), GAITE (store), CHATELET (store), plus Romanian locations
    • Sales Channels: Shopify (e-commerce), API/POS (physical stores), Wholesale
    • Products: ~1,000+ SKUs (Korean skincare products)
    • Integrations: Shopify Hub for e-commerce sync, custom API scripts for POS/wholesale orders

    Key Terminology

    • SKU (Stock Keeping Unit): Unique product code (e.g., SERUM-001)
    • Purchase Order (PO): Order placed with a supplier to buy inventory
    • Sales Order (SO): Order from a customer (Shopify, POS, wholesale)
    • Stock Adjustment: Manual correction to inventory for damage, shrinkage, or count discrepancies
    • Warehouse Transfer: Movement of stock between locations
    • Sales Group: Channel or source for reporting (Shopify, API, Wholesale)

    Product Management

    ▼

    Products in Unleashed include all items sold by Feel My Skin: serums, creams, sets, etc. Each product record contains pricing, supplier info, cost allocation, and inventory tracking.

    Step 1: Access Product Management

    From the main menu, click Products. You will see the full list of all active products.

    Products list

    Step 2: View Product Details

    Click on the product name to view its details: SKU code, purchase price, selling price, description, images, suppliers.

    Product details

    Key Information for Each Product

    • SKU (Stock Keeping Unit): Unique code (e.g., SERUM-001) – must match Shopify product ID
    • Barcode: Barcode for quick scanning in POS/stores
    • Unit of Measure: Unit (piece, box, carton)
    • Cost Price: Purchase price from the supplier
    • Retail Price (Default Sale Price): Customer selling price used by default
    • Supplier: Default supplier for this product
    • Reorder Level: Minimum stock before automatic reordering alert
    • Product Group: Category for reporting (e.g., "Serums", "Creams", "Sets")
    • Cost Distribution Method: MUST be set to "by base quantity" (NOT "by value") – this ensures COGS is calculated correctly

    Step 3: Add a New Product

    Click + New Product. Fill in the required fields: name, SKU, purchase price, selling price, default supplier, and cost distribution method.

    Batch Changing Product Codes

    To change multiple product SKUs at once (e.g., syncing with Shopify IDs):

    1. Go to Products menu
    2. Click Change Codes (in the toolbar or Inventory section)
    3. Select Batch to import a template
    4. Upload your CSV with old code, new code, and product names
    5. Verify and click Import

    This is much faster than changing codes one by one and is essential for syncing Shopify SKUs into Unleashed.

    🔴
    CRITICAL: Cost Distribution Setting
    The cost distribution method MUST be set to "by base quantity", NOT "by value". This fixes inflated COGS calculations. If you see abnormally high COGS or margin issues, check this setting first.
    ⚠
    Warning: Only modify the SKU or product code if you are certain. Changing codes affects all historical orders and Shopify sync. Use batch changes rather than individual edits to maintain consistency.

    Customer & Supplier Setup

    ▼

    Customers and Suppliers are essential for creating Sales Orders (SOs) and Purchase Orders (POs). Proper setup ensures accurate order fulfillment and reporting.

    Customer Setup

    Customer Codes Strategy

    Feel My Skin uses a practical approach to customer management:

    • Channel customers (one per channel): "Shopify", "Clichy POS", "Gaite POS", "Chatelet POS", "Wholesale"
    • Individual wholesale customers: Only create individual customer records for B2B wholesale accounts
    • Do NOT create individual customers for Shopify or POS: These are automatically grouped by channel

    Creating a Customer

    1. Menu → Customers → + New Customer
    2. Enter Customer Code (e.g., "SHOPIFY", "GAITE-POS", "ACME-WHOLESALE")
    3. Enter Customer Name (e.g., "Shopify E-Commerce", "Gaite Store", "ACME Wholesale")
    4. Fill in contact details (email, phone, address) if applicable
    5. Set default currency and payment terms
    6. Click Save

    Importing Customers in Bulk

    If you have a spreadsheet of customers, use the import feature:

    1. Go to Import/Export → Customers
    2. Download the template (CSV)
    3. Fill in: Customer Code, Customer Name, Email, Phone, Address, City, Country
    4. Upload the file back into Unleashed

    Supplier Setup

    Supplier Codes Strategy

    Create one record per supplier company. Include standard information:

    • Supplier Code: Unique identifier (e.g., "AMOREPACIFIC", "ETUDE")
    • Supplier Name: Full company name
    • Contact Details: Email, phone, address
    • Default Warehouse: Where POs from this supplier typically arrive (e.g., DEPOT_FEEL)
    • Lead Time: Number of days from order date to expected delivery
    • Default Notes: Payment terms, order minimums, contact person, etc.

    Creating a Supplier

    1. Menu → Suppliers → + New Supplier
    2. Enter Supplier Code and Supplier Name
    3. Set Default Warehouse (where stock typically arrives)
    4. Enter Lead Time in days (e.g., 10 days for Korean suppliers)
    5. Add contact details and any default notes
    6. Click Save

    Why Lead Times Matter

    When you create a Purchase Order with a supplier that has a lead time set, Unleashed automatically calculates the expected delivery date. For example:

    • Order Date: January 1
    • Lead Time: 10 days
    • Expected Delivery: January 11

    This helps you plan stock arrivals and avoid stockouts. Always verify and update lead times as supplier relationships change.

    ✓
    Tip: Set default warehouses on supplier records so that POs automatically route stock to the correct location. This saves time on every order.

    Pricing Configuration

    ▼

    Unleashed supports a flexible pricing hierarchy. Different customers and channels can have different prices for the same product.

    Pricing Hierarchy (Top to Bottom)

    Unleashed applies prices in this order. The first match wins:

    1. Customer-Specific Pricing – Custom price for this exact customer
    2. Sale Price Tier – Based on quantity ordered (e.g., buy 10+ units = discount)
    3. Default Sale Price – The default price on the product record

    Setting Customer-Specific Pricing

    For wholesale customers with negotiated prices:

    1. Go to Customers and select the customer
    2. Click on the Customer Pricing section
    3. Add a line with:
      • Product or Product Group (e.g., "SERUM-001" or "Serums")
      • Price Type – "Fixed Price" (e.g., £10) or "Discount Percentage" (e.g., 20%)
      • Valid From / Valid To – Optional date range
    4. Click Save

    Bulk Importing Customer Pricing

    To set prices for many products at once:

    1. Go to Import/Export → Customer Pricing
    2. Download the template
    3. Fill in:
      • Customer Code (e.g., "ACME-WHOLESALE")
      • Product Group (optional – applies to all products in the group)
      • Price/Discount (e.g., "50" for fixed price or "20%" for discount)
      • Valid From / Valid To (e.g., "2026-01-01" to "2026-06-30")
    4. Upload and confirm

    Managing Price Reviews

    For wholesale customers, it's common to review prices quarterly or annually. Use the "Valid To" date to:

    • Set expiry dates for negotiated prices
    • Create automatic reminders to renew pricing agreements
    • Track when prices go back to default after the "Valid To" date

    To extend pricing, update the "Valid To" date and re-save.

    ⚠
    Important: Use Product Groups for bulk pricing changes. If you apply pricing to a product group, ALL products in that group automatically inherit the pricing. This saves time but requires careful planning.

    Shopify Integration

    ▼

    The Shopify Hub is Unleashed's connection to your e-commerce store. It automatically syncs products and imports sales orders from Shopify.

    How Shopify Integration Works

    Step 1: Setting Up the Shopify Hub

    The Shopify Hub must be installed and configured before orders can sync:

    1. Menu → Integrations → Shopify
    2. Click Configure
    3. Log in to your Shopify account and authorize the Unleashed app
    4. Configure settings (see below)

    Step 2: Key Configuration Settings

    When configuring the Shopify Hub, pay attention to these critical settings:

    • Default Warehouse for Shopify Orders: Set to "Shopify" or your e-commerce fulfillment location
    • Create Missing Products: Enable this so new Shopify products automatically create in Unleashed (on first order)
    • Create Missing Customers: Typically enabled – creates "Shopify" as the customer
    • SKU Matching: Ensure Shopify SKUs match Unleashed product codes (see SKU Sync below)
    • Sales Group Assignment: Set to "Shopify" for reporting/analytics

    Product SKU Matching (Critical!)

    The SKU Sync Challenge

    For Shopify orders to match to the correct products in Unleashed, the Shopify product SKU must exactly match the Unleashed product code. This requires coordination:

    • Shopify generates product IDs (SKUs) – these may be internal numbers
    • Unleashed product codes are your internal inventory codes (e.g., "SERUM-001")
    • If they don't match, orders come through with no product assigned

    Solution: Batch SKU Sync

    To align Shopify and Unleashed SKUs:

    1. Export your product catalog from Shopify (via Shopify admin)
    2. In Unleashed, go to Products → Change Codes → Batch
    3. Upload a CSV with:
      • Current Product Code (your Unleashed code)
      • New Product Code (from Shopify SKU)
    4. Verify the changes and import

    After this, new Shopify orders will correctly match to Unleashed products.

    Tax Rate Configuration (CRITICAL for Shopify Sync)

    Understanding the Tax Issue

    Shopify orders include tax rates (20%, 5.5%, etc.), but Unleashed may not recognize these tax rates if they're not configured. When orders sync, Unleashed looks for matching tax rates. If none exist, the order fails to import.

    Setting Up Tax Rates

    For Feel My Skin, you must configure these tax rates:

    1. Menu → Settings → Taxes
    2. Click + New Tax Rate for each rate you need
    3. Add 20% – for standard VAT in France
    4. Add 5.5% – for reduced VAT (or whatever Shopify uses)
    5. Keep the system default at 0% (no tax) for POS/wholesale orders
    6. For each rate, enter a Tax Code (e.g., "20% Sales" or "5.5% Reduced")
    7. Tick Sales Tax checkbox for each
    8. Click Save
    🔴
    CRITICAL: Tax Rate Setup
    NEVER remove the 0% tax rate (the system default). Keep it in place even after adding 20% and 5.5%. The 0% rate is essential for POS/wholesale orders which don't carry tax in Unleashed. Shopify orders use 20%/5.5%, while POS orders use 0%.

    Shopify Order Flow

    How Orders Appear in Unleashed

    When a customer buys from your Shopify store:

    1. Order appears in Shopify with status "Pending" (not yet fulfilled)
    2. Unleashed receives the order and creates a Sales Order with status "Parked" or "Placed"
    3. When you fulfill the order in Shopify, it updates in Unleashed to "Shipped" or "Completed"

    Fulfillment Workflow

    There are two ways to fulfill Shopify orders:

    • Fulfill in Shopify: Mark items as "Fulfilled" in Shopify → Unleashed auto-updates
    • Fulfill in Unleashed: Complete the Sales Order in Unleashed → Optionally sync back to Shopify

    Recommendation: Fulfill in Shopify first, then Unleashed auto-completes when the order status changes.

    Sales Groups for Channel Analytics

    All Shopify orders are tagged with Sales Group: Shopify. This allows you to filter reports and dashboards by channel. You can also create sales groups for other channels:

    • Shopify – E-commerce orders
    • API/POS – Physical store orders imported via script
    • Wholesale – B2B orders

    Use these groupings to track revenue, margin, and inventory by channel.

    Troubleshooting Shopify Sync

    Orders Not Syncing?

    Common issues and solutions:

    • Missing tax rates: Check Settings → Taxes. Add 20% and 5.5% rates if missing.
    • SKU mismatch: Verify Shopify SKU matches Unleashed product code. Use batch change to sync.
    • Connection error: Menu → Integrations → Shopify → Check Connection Log for errors.
    • Products not created: Ensure "Create Missing Products" is enabled in Shopify config.

    Importing Missing Orders Manually

    If orders don't auto-import, use the manual import feature:

    1. Go to Integrations → Shopify
    2. Click Import Missing Sales Orders
    3. Set the date range (e.g., "Jan 1 to Today")
    4. Click Import
    5. Check the Connection Log for any errors
    ✓
    Tip: After fixing tax rates or SKU issues, always test with a recent Shopify order to confirm the sync works before processing hundreds of historical orders.

    Purchase Orders

    ▼

    Purchase orders (POs) allow us to order stock from our suppliers. Properly managing POs ensures timely stock arrivals and accurate inventory levels.

    Step 1: Access Purchase Orders

    Menu → Purchase Orders to view all POs. You can filter by status: Open, Placed, Received, Completed.

    Purchase orders list

    Step 2: Create a New PO

    Click on + New Purchase Order:

    1. Select the Supplier (e.g., "AMOREPACIFIC")
    2. Select the destination Warehouse (e.g., DEPOT_FEEL)
    3. Add product lines with quantities and unit prices
    4. Verify the lead time (auto-calculated from supplier record)
    5. Check the total cost
    6. Click Save
    Create purchase order

    Step 3: Place the Order

    Once saved, click Place Order to send the PO to the supplier. The status changes to "Placed".

    Step 4: Receive Stock

    When the order arrives, click Receive. Unleashed then creates a receipt to record incoming stock:

    1. Verify the quantities match the order
    2. Note any discrepancies (e.g., "5 units missing")
    3. Click Confirm

    Once confirmed, stock is added to the warehouse and the PO moves to "Received" or "Completed" status.

    Supplier Lead Times on POs

    Expected Delivery Date

    When you create a PO, Unleashed calculates the expected delivery date automatically:

    • If the supplier has a lead time set (e.g., 10 days), the delivery date is: Order Date + 10 days
    • If no lead time is set, the delivery date defaults to the order date (you must fill it in manually)

    This helps you plan stock availability and forecast when POs will arrive.

    Importing Missing Orders from Suppliers

    Backfilling Historical POs

    If you need to import past orders (e.g., orders from Shopify that triggered POs in Unleashed):

    1. Go to Integrations (if applicable) or manually create POs
    2. Use Import/Export → Purchase Orders for bulk import
    3. Create a CSV with: Supplier Code, Product Code, Quantity, Unit Price, Warehouse
    4. Upload and confirm
    🔴
    CRITICAL: Back-Order Bug Warning
    DO NOT create a Purchase Order from within a Sales Order. There is a known bug: if you click "Create PO" on a Sales Order, it creates a linked PO that can cause the stock to be double-counted (once in the SO, once in the PO), resulting in inflated inventory. Instead, create POs manually in the PO section or through the purchase menu. If you accidentally create a PO this way, use Transaction Inquiry to identify and delete the duplicate stock movement.
    ✓
    Tip: Always verify received quantities against the packing slip or invoice. If there's a discrepancy (e.g., 5 units short), note it in the receipt comments and follow up with the supplier.

    Sales Orders

    ▼

    Sales orders (SOs) record sales to B2C and B2B customers. They track customer orders, inventory allocation, and fulfillment status.

    Step 1: Access Sales Orders

    Menu → Sales Orders to view all SOs. Filter by status: Open, Placed, Backordered, Shipped, Completed.

    Sales orders list

    Step 2: Create a New SO (Manual Order)

    For wholesale or manual orders (Shopify orders come in automatically):

    1. Click + New Sales Order
    2. Select the Customer (e.g., "Shopify", "ACME-WHOLESALE")
    3. Select the Warehouse for shipping
    4. Add product lines with quantities
    5. Check stock availability (green = available, red = out of stock)
    6. Set the desired delivery date
    7. Assign a Sales Group (e.g., "Shopify", "API/POS", "Wholesale") for reporting
    8. Click Save
    Create sales order

    Step 3: Sales Groups for Analytics

    Always assign a Sales Group to each order. This enables reporting by channel:

    • Shopify – E-commerce orders (auto-assigned for Shopify orders)
    • API/POS – Physical store orders from POS script
    • Wholesale – B2B wholesale orders

    These groups are used in dashboards and reports to analyze revenue, margin, and inventory by channel.

    Step 4: Prepare and Fulfill the Order

    When the order is ready to ship:

    1. In the SO, click Dispatch or Ship (depending on Unleashed version)
    2. Verify quantities and confirm the shipment
    3. Unleashed records the stock departure and generates a shipping document
    4. Update status to "Shipped" or "Completed"

    Shopify Order Status Mapping

    Shopify orders appear in Unleashed with these statuses:

    • Parked/Placed: Order received from Shopify, awaiting fulfillment
    • Backordered: Partial shipment sent, balance awaiting stock (rare – avoid manually creating POs from SO)
    • Shipped/Completed: Order fulfilled and marked complete in Unleashed

    Difference Between Fulfilling in Shopify vs Unleashed

    Best Practice: Fulfill in Shopify First

    The recommended workflow is:

    1. In Shopify: Pack the order, print the label, mark items as "Fulfilled" in Shopify
    2. In Unleashed: The order status automatically updates (usually to Shipped/Completed)
    3. Stock is automatically deducted from inventory when Shopify sync processes the fulfillment
    ⚠
    Important: Always verify that stock is available before creating a SO. If stock is insufficient, the order may be set to "Backordered", which complicates month-end closing. Check the stock indicator (green = available, red = short) before confirming the order.

    Stock Adjustments

    ▼

    Adjustments allow you to correct counting errors, record breakages, or stock losses. They're essential for maintaining accurate inventory levels.

    Step 1: Access Adjustments

    Menu → Stock → Stock Adjustments to view all adjustments.

    Stock adjustments

    Step 2: Create a New Adjustment

    Click + New Adjustment:

    1. Select the Warehouse (e.g., DEPOT_FEEL, CLICHY)
    2. Add adjustment lines:
      • Select the Product
      • Enter the Quantity (positive = add stock, negative = remove stock)
      • Select a Reason (see Common Adjustment Reasons below)
    3. Add notes if needed (e.g., "Found broken serum bottles in Clichy store")
    4. Click Save

    Common Adjustment Reasons

    • Inventory Count: Discrepancy found during a physical count
    • Damage: Damaged products discovered (broken bottles, expired, etc.)
    • Shrinkage: Unexplained loss (theft, evaporation, etc.)
    • Stock Transfer: Manual transfer between warehouses (prefer using Warehouse Transfers instead)
    • Sample: Taken for demonstration or staff testing
    • Return: Customer return not recorded through the SO system
    • Recount: Correction after a recount confirmed the discrepancy

    Month-End Stock Adjustments

    Handling Stock Shortfalls at Month-End

    When closing a month in Unleashed, you may find that stock levels don't match orders. This is common with POS orders and physical stores. Solution:

    1. Try to bulk-complete all SOs for the month
    2. If the system shows "insufficient stock", note which products are short
    3. Create a stock adjustment with reason "Inventory Count" for each shortfall
    4. Re-try the bulk completion

    Example: "API channel sold 5 units of SERUM-001 but Unleashed only had 3 on hand. Create adjustment: +2 units, reason: Inventory Count"

    ⚠
    Warning: Stock adjustments directly affect inventory levels and reports. Always record a clear reason for traceability. If you adjust stock incorrectly, it skews reports and COGS calculations.

    Simplified Warehouse Strategy

    Shopify Default Warehouse → Depot Fill Strategy

    To avoid constant stock transfers, Feel My Skin uses this simplified approach:

    • Shopify orders: All route to the "Shopify" default warehouse (set in Shopify Hub config)
    • Depot Fill transfers: Periodically move stock from DEPOT_FEEL to the Shopify warehouse in one bulk transfer
    • Physical stores: Stock is held at each store location and managed via POS/API orders

    This reduces the number of transfers and makes inventory easier to track.

    ✓
    Tip: Schedule transfers regularly (e.g., weekly or bi-weekly) to balance stock between DEPOT_FEEL, the three Paris stores, and the Romanian warehouses. Use the Reorder Report to see low stock locations and plan transfers.

    Month-End Close

    ▼

    Month-end close is a critical process. It finalizes all transactions, ensures inventory accuracy, and closes the accounting period so reports reflect final numbers.

    Step-by-Step Month-End Process

    Step 1: Complete All Open Sales Orders (Bulk)

    In the last days of the month, complete all SOs:

    1. Menu → Sales Orders
    2. Filter by Status: Open or Placed
    3. Select ALL orders (use the checkbox at the top to select all on page, repeat for all pages)
    4. Click Bulk Actions → Complete
    5. Click Process

    Expected Outcome: System processes all orders and updates stock. Some may succeed, some may fail if stock is insufficient.

    Step 2: Handle Failed Orders (Stock Adjustments)

    For orders that failed to complete due to insufficient stock:

    1. Identify which products are short (system will list them)
    2. For each shortfall, create a Stock Adjustment:
      • Go to Stock → Stock Adjustments
      • Create an adjustment with Reason: Inventory Count
      • Add the missing units (e.g., if SERUM-001 needs 3 more units, adjust +3)
      • Save
    3. Re-run the bulk complete for the failed orders

    Repeat until all orders are completed successfully.

    Step 3: Verify Stock Reconciliation

    After completing all orders, verify that physical stock matches Unleashed:

    1. Run Stock on Hand Inquiry for each warehouse
    2. Compare Unleashed numbers to your physical count (if available)
    3. If discrepancies exist, create adjustments with reason "Recount" to correct them

    Step 4: Close the Period (Accounting)

    Once all orders are completed and stock is reconciled, close the period:

    1. Menu → Settings → Period Management (or similar, depending on your Unleashed version)
    2. Select the month/period to close (e.g., "January 2026")
    3. Click Close Period
    4. Confirm that you want to finalize this period (typically can't edit transactions after closing)

    Step 5: Generate Final Reports

    Once the period is closed, run month-end reports for accounting/finance:

    • Product Margin Inquiry – Gross profit by product and channel
    • Costing Analysis Report – Total COGS and inventory value
    • Stock on Hand Inquiry – Final inventory levels by warehouse
    • Export to CSV and provide to your finance/accounting team

    Common Month-End Issues & Solutions

    Issue: "Insufficient Stock" Errors When Bulk Completing Orders

    Cause: POS orders and physical store orders have imprecise stock tracking. The system shows fewer units on hand than actually sold.

    Solution: Create stock adjustments to correct the difference and re-run the bulk complete.

    Issue: COGS Appears Inflated

    Cause: Cost distribution method is set to "by value" instead of "by base quantity".

    Solution: Check each product's cost distribution method. Change to "by base quantity" and recalculate.

    Issue: Stock Discrepancies Persist

    Cause: Missing orders, incorrect POS imports, or duplicate stock transfers.

    Solution: Use Transaction Inquiry to trace stock movements and identify the source of the discrepancy.

    ⚠
    Best Practice: Plan for 2-3 hours for month-end close, especially in the first few months. The process will become faster once you understand your data patterns and stock accuracy improves.
    ✓
    Tip: After closing a period, create a backup or snapshot of the data. This protects against accidental changes and provides an audit trail.

    Daily Checks

    ▼

    Each day, perform these checks to maintain system health and catch problems early.

    Morning: Check Pending Orders (5 minutes)

    Open the Dashboard and review:

    • Sales orders in "Open" or "Placed" status ready to be shipped (especially Shopify)
    • Purchase orders awaiting receipt (check expected arrival dates)
    • Low stock alerts (products approaching reorder level)
    • Any backorders or stuck transfers
    Daily dashboard view

    Midday/Afternoon: Confirm Receipts (5-10 minutes)

    Check Purchase Orders in "Placed" status. If goods have arrived:

    1. Verify the delivery against the PO
    2. Click Receive on the PO
    3. Confirm quantities and note any discrepancies
    4. Click Confirm

    End of Day: Close Orders & Check Shipments (10 minutes)

    Before leaving for the day:

    1. For Shopify orders that were fulfilled in Shopify, verify Unleashed status updated to "Shipped" or "Completed"
    2. For POS/retail orders, ensure they're marked as shipped/completed
    3. Confirm no orders are stuck in "Parked" or "Placed" status unexpectedly
    End of day checks

    Weekly: Stock Report (Friday, 15 minutes)

    Every Friday, generate a Stock Report to assess inventory health:

    1. Run Stock on Hand Inquiry for all warehouses
    2. Identify:
      • Low or negative stock
      • Excess inventory (overstocked items)
      • Products approaching expiration (if tracked)
    3. Create POs for low stock items
    4. Plan warehouse transfers to balance stock between locations

    • ✓ No orders in "Backorder" status for more than 3 days (escalate if stuck)
    • ✓ Minimum stock (Reorder Level) reached → Create a PO immediately
    • ✓ All warehouse transfers confirmed (Received status)

    Tip: Spend just 5-10 minutes each morning on the dashboard. This early detection prevents stockouts, missed orders, and month-end surprises.

    Critical Rules

    ▼

    These rules must always be followed to avoid costly mistakes and maintain data integrity.

    🔴
    Rule 1: Cost Distribution = "By Base Quantity" (NOT "By Value")
    Every product MUST have cost distribution set to "by base quantity". Using "by value" inflates COGS and distorts margin analysis. Check this setting regularly, especially for imported products.
    🔴
    Rule 2: NEVER Remove Tax Rate 0%
    Keep the 0% tax rate in Settings → Taxes. It's essential for POS/wholesale orders which don't carry tax. Adding 20% and 5.5% for Shopify is fine, but NEVER delete the 0% default.
    🔴
    Rule 3: Never Create PO from Sales Order (Back-Order Bug)
    DO NOT click "Create PO" within a Sales Order. This creates a linked PO that doubles inventory counts. Create POs separately in the PO menu. If you accidentally do this, use Transaction Inquiry to delete the duplicate.
    🔴
    Rule 4: Check Stock Before Selling
    Before creating an SO (or accepting a Shopify order), verify stock is available in the correct warehouse. Unavailable stock creates backlogs and complicates fulfillment.
    🔴
    Rule 5: Batch Change Product SKUs, Don't Change One by One
    If syncing Shopify SKUs into Unleashed, use the Batch Change feature (Products → Change Codes → Batch). This is faster and prevents individual errors.
    🔴
    Rule 6: Record Reasons for All Adjustments
    Every stock adjustment (Damage, Shrinkage, Inventory Count, etc.) must have a documented reason. This ensures traceability and audit compliance.
    🔴
    Rule 7: Confirm Receipts Promptly
    As soon as supplier stock is received, click "Receive" in the PO. This updates stock levels in real time and prevents inventory discrepancies.
    🔴
    Rule 8: Use Correct Warehouse Codes
    Always use the correct warehouse (DEPOT_FEEL, CLICHY, GAITE, CHATELET, etc.). A wrong selection creates discrepancies and breaks reporting.
    🔴
    Rule 9: Never Delete an Order
    If an order is incorrect, mark it as "Cancelled" or create a note. Never delete it, as this loses history and makes auditing impossible.
    🔴
    Rule 10: Check Prices Regularly
    Purchase and selling prices must be kept up to date. Outdated prices lead to incorrect margins and inaccurate POs.
    🔴
    Rule 11: Sync Shopify Tax Rates Before Importing Orders
    Create 20% and 5.5% tax rates in Settings → Taxes BEFORE importing Shopify orders. Without matching tax rates, orders fail to import.
    🔴
    Rule 12: When in Doubt, Contact a Manager
    If something seems abnormal (negative stock, inconsistent price, stuck transfer, sync errors), do not try to fix it alone. Report it immediately to your manager or the Unleashed implementation team.

    Troubleshooting Table

    Problem Cause Solution
    Shopify orders not syncing Missing or incorrect tax rates in Settings Go to Settings → Taxes. Add 20% and 5.5% tax rates. Ensure 0% default is not removed. Re-import orders.
    COGS looks inflated / Margins are wrong Cost distribution method set to "by value" instead of "by base quantity" Check each product: Products → [Product] → Cost Distribution Method. Change to "by base quantity". Recalculate.
    Stock count doubled / Backordered with doubled inventory PO was created from within Sales Order (back-order bug) Use Transaction Inquiry to find the duplicate stock movement. Delete or reverse it. Never create POs from within SO.
    Negative stock (-1, -5, etc.) Orders completed without sufficient inventory Create a Stock Adjustment with reason "Inventory Count" to correct. Check the movement history to understand cause.
    Order stuck / Cannot save Required fields missing or data validation error Verify all required fields are filled. Check for typos in customer/product codes. Contact support if persists.
    Transfer not received / Stuck in transit Warehouse Transfer not finalized Go to Warehouse Transfers, find the transfer, click "Receive" to finalize. Verify quantities match.
    Stock differs from physical count Unrecorded sales, damage, theft, or counting error Create a Stock Adjustment with reason "Inventory Count" to record the difference. Investigate the cause separately.
    Supplier not found Supplier not created in Unleashed Check the spelling. If missing, create a new supplier: Menu → Suppliers → + New Supplier. Set default warehouse and lead time.
    Customer not found Customer record doesn't exist Create the customer: Menu → Customers → + New Customer. Use simple codes like "SHOPIFY", "GAITE-POS", "ACME-WHOLESALE".
    Shopify SKU mismatch (products not matching) Shopify SKU doesn't match Unleashed product code Use Products → Change Codes → Batch to sync Shopify SKUs. Download template, import from Shopify export, bulk update.
