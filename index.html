    • Introduction

    ▼

    Bienvenue dans le guide de gestion des stocks Unleashed de Feel My Skin. Ce guide est votre référence complète pour gérer les stocks, les clients, les fournisseurs, les commandes et les rapports dans Unleashed.

    Qu'est-ce que Unleashed ?

    Unleashed est un système de gestion des stocks basé sur le cloud qui suit les stocks dans plusieurs entrepôts, gère les commandes de vente et d'achat, et fournit des rapports et analyses détaillés. Pour Feel My Skin, il s'intègre avec Shopify (e-commerce), les systèmes de point de vente (magasins de détail) et les systèmes comptables.

    Configuration actuelle de Feel My Skin

    • Entrepôts: DEPOT_FEEL (main), CLICHY (store), GAITE (store), CHATELET (store), plus des entrepôts en Roumanie
    • Canaux de vente : Shopify (e-commerce), API/PDV (magasins physiques), Vente en gros
    • Produits : ~1 000+ SKU (produits de soin coréens)
    • Intégrations : Shopify Hub pour la sync e-commerce, scripts API personnalisés pour commandes PDV/gros

    Terminologie clé

    • SKU (numéro de gestion des stocks) : Code produit unique (ex. : SERUM-001)
    • Bon de commande (BC) : Commande passée auprès d'un fournisseur pour acheter des stocks
    • Bon de commande client : Commande d'un client (Shopify, PDV, vente en gros)
    • Ajustement des stocks : Correction manuelle pour dommages, manques ou écarts de comptage
    • Transfert d'entrepôt : Mouvement de stock entre emplacements
    • Groupe de ventes : Canal ou source de reporting (Shopify, API, Vente en gros)
    ✓
    Conseil : Utilisez la zone de recherche dans la barre latérale pour trouver rapidement des sujets dans ce guide.

    Aperçu du tableau de bord

    ▼

    Le tableau de bord est votre centre de commande. Il affiche un résumé des indicateurs clés, des tâches en attente et des alertes en temps réel.

    Accéder au tableau de bord

    Dans le menu principal, cliquez sur Tableau de bord. Vous verrez un aperçu personnalisable avec des widgets affichant :

    • Chiffre d'affaires total et marge brute
    • Produits et catégories les plus performants
    • Alertes de stock bas
    • Commandes en attente et livraisons reçues
    • Tendances des ventes par canal

    Routine quotidienne du tableau de bord

    Chaque matin, consacrez 5 minutes au tableau de bord pour :

    1. Vérifier les alertes de stock bas – identifier les produits à réapprovisionner
    2. Consulter les commandes de vente en attente – voir quelles commandes attendent d'être expédiées
    3. Vérifier les livraisons reçues – confirmer l'arrivée des livraisons fournisseurs
    4. Surveiller les commandes en souffrance – identifier les commandes à livraison partielle

    Personnaliser votre tableau de bord

    Cliquez sur Modifier le tableau de bord pour personnaliser votre vue :

    • Ajouter ou supprimer des widgets
    • Changer le type de widget (graphique, grille, carte KPI)
    • Filtrer par entrepôt, canal de vente ou groupe de produits
    • Définir des plages de temps (quotidien, hebdomadaire, mensuel)
    ✓
    Conseil : Créez des tableaux de bord distincts pour les différents rôles (par exemple, le tableau de bord du responsable de magasin, le tableau de bord des finances) pour des flux de travail quotidiens plus rapides.

    Navigation et recherche

    ▼

    Unleashed dispose d'une structure de menu cohérente et d'outils de recherche puissants. Les maîtriser vous fera gagner un temps considérable chaque jour.

    Structure du menu principal

    • Achats : Créer et gérer les bons de commande
    • Stocks : Gérer les produits, niveaux de stock et entrepôts
    • Ventes : Créer et gérer les commandes de vente
    • Rapports: Run inventory, sales, and purchase reports
    • Paramètres : Configurer les options système, taux de TVA et préférences

    Barre d'actions rapides

    La barre d'actions rapides (généralement à gauche) fournit des raccourcis vers les tâches courantes : créer un BC, voir le stock disponible, lancer des rapports. Utilisez ces raccourcis plutôt que de naviguer dans le menu complet.

    Rechercher des produits

    Vous pouvez rechercher des produits de deux façons :

    1. Utiliser la barre de recherche globale en haut de l'écran — tapez n'importe quelle partie du nom ou du code produit
    2. Aller dans Stocks → Produits et utiliser les filtres de colonnes pour affiner les résultats
    ✓
    Conseil : Utilisez le préfixe FMSK_ pour rechercher des produits rapidement. Tous les codes de produits Feel My Skin commencent par FMSK_ (par exemple, FMSK_11691). C'est beaucoup plus rapide que de rechercher par nom de produit.

    Gestion des produits

    ▼

    Les produits dans Unleashed comprennent tous les articles vendus par Feel My Skin : sérums, crèmes, coffrets, etc. Chaque produit dispose de plusieurs onglets contenant les prix, informations fournisseurs, données de coût et suivi des stocks. FMS utilise le préfixe FMSK_ pour tous les codes produits (ex. : FMSK_11691).

    Créer un nouveau produit (flux Shopify en premier)

    Chez FMS, les nouveaux produits sont d'abord créés sur Shopify, puis enregistrés dans Unleashed :

    1. Créer le produit sur Shopify en premier — ajouter nom, images, prix
    2. Shopify génère un code produit — copier ce code
    3. Aller dans Stocks → Produits → Ajouter un produit dans Unleashed
    4. Saisir le code généré par Shopify comme code produit (format FMSK_XXXXX)
    5. Saisir la description du produit
    6. Le code-barres peut être ajouté plus tard — une fois le produit arrivé au dépôt, le scanner et l'ajouter à la fiche produit
    7. Par défaut, les produits sont configurés comme vendables avec type décroissant
    8. Enregistrer le produit
    ⚠
    Important : Créez toujours le produit sur Shopify en premier pour obtenir le code. Ne créez pas les produits dans Unleashed en premier - cela cause une incompatibilité des SKU avec la synchronisation Shopify.

    Opérations en lot

    Modification des codes produits en lot

    Pour modifier plusieurs SKU produits en même temps (ex. : synchronisation avec les IDs Shopify) :

    1. Go to Produits menu
    2. Cliquer sur Changer les codes (dans la barre d'outils ou la section Stocks)
    3. Sélectionner Lot pour importer un modèle
    4. Télécharger votre CSV avec l'ancien code, le nouveau code et les noms de produits
    5. Vérifier et cliquer sur Importer

    C'est beaucoup plus rapide que de modifier les codes un par un et indispensable pour synchroniser les SKUs Shopify dans Unleashed.

    ⚠
    Avertissement : Ne modifiez le SKU ou le code produit que si vous en êtes certain. La modification des codes affecte toutes les commandes historiques et la synchronisation Shopify. Utilisez les modifications en lot plutôt qu'individuelles pour maintenir la cohérence.

    Configuration des clients et fournisseurs

    ▼

    Les clients et fournisseurs sont indispensables pour créer des commandes de vente (CV) et des bons de commande (BC). Une configuration correcte garantit un traitement précis des commandes et des rapports fiables.

    Configuration des clients

    Stratégie de codes clients

    • Clients par canal (un par canal) : «Shopify», «Clichy PDV», «Gaite PDV», «Chatelet PDV», «Vente en gros»
    • Clients gros individus : Créer des fiches individuelles uniquement pour les comptes B2B en gros
    • NE PAS créer de clients individuels pour Shopify ou PDV : Ils sont automatiquement regroupés par canal

    Créer un client

    1. Menu → Clients → + Nouveau client
    2. Saisir le code client (ex. : «SHOPIFY», «GAITE-PDV», «ACME-WHOLESALE»)
    3. Saisir le nom du client (ex. : «Shopify E-Commerce», «Boutique Gaite», «ACME Wholesale»)
    4. Remplir les coordonnées (email, téléphone, adresse) si applicable
    5. Définir la devise par défaut et les conditions de paiement
    6. Click Save

    Importer des clients en masse

    Si vous disposez d'un tableau de clients, utilisez la fonction d'importation :

    1. Aller dans Import/Export → Clients
    2. Télécharger le modèle (CSV)
    3. Remplir : Code client, Nom du client, Email, Téléphone, Adresse, Ville, Pays
    4. Télécharger le fichier à nouveau dans Unleashed

    Configuration des fournisseurs

    Liste des fournisseurs dans Unleashed

    Stratégie de codes fournisseurs

    Créer une fiche par société fournisseur. Inclure les informations standard :

    • Code fournisseur : Identifiant unique (ex. : «AMOREPACIFIC», «ETUDE»)
    • Nom du fournisseur : Nom complet de la société
    • Coordonnées : Email, téléphone, adresse
    • Entrepôt par défaut : Où arrivent habituellement les BC de ce fournisseur (ex. : DEPOT_FEEL)
    • Délai de livraison : Nombre de jours entre la date de commande et la livraison prévue
    • Notes par défaut : Conditions de paiement, minimum de commande, personne de contact, etc.

    Créer un fournisseur

    1. Menu → Fournisseurs → + Nouveau fournisseur
    2. Saisir le code fournisseur et le nom du fournisseur
    3. Définir l'entrepôt par défaut (où le stock arrive habituellement)
    4. Saisir le délai de livraison en jours (ex. : 10 jours pour les fournisseurs coréens)
    5. Ajouter les coordonnées et toute note par défaut
    6. Click Save

    Pourquoi les délais de livraison sont importants

    Quand vous créez un bon de commande avec un fournisseur ayant un délai défini, Unleashed calcule automatiquement la date de livraison prévue. Par exemple :

    • Date de commande : 1er janvier
    • Délai de livraison : 10 jours
    • Livraison prévue : 11 janvier

    Cela vous aide à planifier les arrivées de stock et éviter les ruptures. Vérifiez et mettez toujours à jour les délais lorsque les relations fournisseurs évoluent.

    ✓
    Conseil : Définissez des entrepôts par défaut sur les fiches fournisseurs pour que les BC routent automatiquement le stock au bon emplacement. Cela fait gagner du temps sur chaque commande.

    Configuration des prix

    ▼

    Unleashed prend en charge une hiérarchie de prix flexible. Différents clients et canaux peuvent avoir des prix différents pour le même produit.

    Hiérarchie des prix (du plus au moins prioritaire)

    Unleashed applique les prix dans cet ordre. La première correspondance l'emporte :

    1. Prix spécifique client – Prix personnalisé pour ce client précis
    2. Niveau de prix de vente – Basé sur la quantité commandée (ex. : achat 10+ unités = remise)
    3. Prix de vente par défaut – Le prix par défaut sur la fiche produit

    Définir un prix spécifique client

    Pour les clients gros avec des prix négociés :

    1. Go to Clients and select the customer
    2. Cliquer sur la section Tarification client
    3. Ajouter une ligne avec :
      • Produit ou groupe de produits (ex. : «SERUM-001» ou «Sérums»)
      • Type de prix – «Prix fixe» (ex. : 10 €) ou «Pourcentage de remise» (ex. : 20 %)
      • Valide du / Valide au – Plage de dates optionnelle
    4. Click Save

    Importation en masse des prix clients

    Pour définir des prix pour de nombreux produits en même temps :

    1. Aller dans Import/Export → Tarification client
    2. Télécharger le modèle
    3. Fill in:
      • Code client (ex. : «ACME-WHOLESALE»)
      • Groupe de produits (optionnel – s'applique à tous les produits du groupe)
      • Prix/Remise (ex. : «50» pour prix fixe ou «20 %» pour remise)
      • Valide du / Valide au (ex. : «2026-01-01» au «2026-06-30»)
    4. Télécharger et confirmer

    Gérer les révisions de prix

    Pour les clients gros, il est courant de réviser les prix trimestriellement ou annuellement. Utilisez la date «Valide au» pour :

    • Définir des dates d'expiration pour les prix négociés
    • Créer des rappels automatiques pour renouveler les accords de prix
    • Suivre quand les prix reviennent au défaut après la date «Valide au»

    Pour prolonger les prix, mettez à jour la date «Valide au» et enregistrez.

    ⚠
    Important : Utilisez les groupes de produits pour les changements de prix en masse. Si vous appliquez un prix à un groupe, TOUS les produits de ce groupe héritent automatiquement du prix. Cela fait gagner du temps mais nécessite une planification soigneuse.

    Intégration Shopify

    ▼

    Le Shopify Hub est la connexion d'Unleashed à votre boutique e-commerce. Il synchronise automatiquement les produits et importe les commandes de vente depuis Shopify.

    Fonctionnement de l'intégration Shopify

    Étape 1 : Configurer le Shopify Hub

    Le Shopify Hub doit être installé et configuré avant que les commandes puissent se synchroniser :

    1. Menu → Integrations → Shopify
    2. Click Configure
    3. Se connecter à votre compte Shopify et autoriser l'application Unleashed
    4. Configurer les paramètres (voir ci-dessous)

    Étape 2 : Paramètres de configuration clés

    Lors de la configuration du Shopify Hub, faites attention à ces paramètres critiques :

    • Entrepôt par défaut pour les commandes Shopify : Définir sur «Shopify» ou votre entrepôt e-commerce
    • Créer les produits manquants : Activer pour que les nouveaux produits Shopify se créent automatiquement dans Unleashed (à la première commande)
    • Créer les clients manquants : Généralement activé – crée «Shopify» comme client
    • Correspondance des SKU : S'assurer que les SKU Shopify correspondent aux codes produits Unleashed (voir Sync SKU ci-dessous)
    • Attribution du groupe de ventes : Définir sur «Shopify» pour le reporting/analytique

    Correspondance des SKU produits (Critique !)

    Le défi de la synchronisation des SKU

    Pour que les commandes Shopify correspondent aux bons produits dans Unleashed, le SKU produit Shopify doit correspondre exactement au code produit Unleashed. Cela nécessite une coordination :

    • Shopify génère des IDs produits (SKU) – il peut s'agir de numéros internes
    • Les codes produits Unleashed sont vos codes de stock internes (ex. : «SERUM-001»)
    • S'ils ne correspondent pas, les commandes arrivent sans produit attribué

    Solution : Synchronisation des SKU en lot

    Pour aligner les SKU Shopify et Unleashed :

    1. Exporter votre catalogue produits depuis Shopify (via l'admin Shopify)
    2. In Unleashed, go to Produits → Change Codes → Batch
    3. Upload a CSV with:
      • Code produit actuel (votre code Unleashed)
      • Nouveau code produit (depuis le SKU Shopify)
    4. Vérifier les modifications et importer

    Après cela, les nouvelles commandes Shopify correspondront correctement aux produits Unleashed.

    Configuration des taux de TVA (CRITIQUE pour la sync Shopify)

    Comprendre le problème de TVA

    Les commandes Shopify incluent des taux de TVA (20 %, 5,5 %, etc.), mais Unleashed peut ne pas les reconnaître s'ils ne sont pas configurés. Lors de la synchronisation, Unleashed recherche des taux de TVA correspondants. Si aucun n'existe, la commande échoue à l'importation.

    Configurer les taux de TVA

    Pour Feel My Skin, vous devez configurer ces taux de TVA :

    1. Menu → Settings → Taxes
    2. Click + New Tax Rate for each rate you need
    3. Ajouter 20 % – pour la TVA standard en France
    4. Ajouter 5,5 % – pour la TVA réduite (ou tout taux utilisé par Shopify)
    5. Conserver le défaut système à 0 % (sans taxe) pour les commandes PDV/gros
    6. Pour chaque taux, saisir un code taxe (ex. : «20 % Ventes» ou «5,5 % Réduit»)
    7. Cocher la case Taxe sur les ventes pour chacun
    8. Click Save
    🔴
    CRITIQUE : Configuration des taux de TVA
    NE JAMAIS supprimer le taux de TVA 0 % (le défaut système). Le conserver même après avoir ajouté 20 % et 5,5 %. Le taux 0 % est indispensable pour les commandes PDV/gros qui ne portent pas de taxe dans Unleashed. Les commandes Shopify utilisent 20 %/5,5 %, les commandes PDV utilisent 0 %.

    Flux des commandes Shopify

    Comment les commandes apparaissent dans Unleashed

    Quand un client achète sur votre boutique Shopify :

    1. La commande apparaît dans Shopify avec le statut «En attente» (non encore traitée)
    2. Unleashed reçoit la commande et crée une commande de vente avec le statut «Mise en attente» ou «Placée»
    3. Lorsque vous traitez la commande dans Shopify, elle se met à jour dans Unleashed en «Expédiée» ou «Terminée»

    Flux de traitement des commandes

    Il existe deux façons de traiter les commandes Shopify :

    • Traiter dans Shopify : Marquer les articles comme «Traités» dans Shopify → Unleashed se met à jour automatiquement
    • Traiter dans Unleashed : Compléter la commande de vente dans Unleashed → Synchronisation optionnelle avec Shopify

    Recommandation : Traiter dans Shopify en premier, puis Unleashed se complète automatiquement lorsque le statut de la commande change.

    Groupes de ventes pour l'analytique par canal

    Toutes les commandes Shopify sont taguées avec Groupe de ventes : Shopify. Cela vous permet de filtrer les rapports et tableaux de bord par canal. Vous pouvez aussi créer des groupes de ventes pour d'autres canaux :

    • Shopify – Commandes e-commerce
    • API/PDV – Commandes en magasin physique importées via script
    • Wholesale – B2B orders

    Utilisez ces regroupements pour suivre le chiffre d'affaires, la marge et les stocks par canal.

    Résolution des problèmes de synchronisation Shopify

    Les commandes ne se synchronisent pas ?

    Problèmes courants et solutions :

    • Taux de TVA manquants : Vérifier Paramètres → Taxes. Ajouter les taux 20 % et 5,5 % si manquants.
    • Incompatibilité SKU : Vérifier que le SKU Shopify correspond au code produit Unleashed. Utiliser la modification en lot pour synchroniser.
    • Erreur de connexion : Menu → Intégrations → Shopify → Vérifier le journal de connexion pour les erreurs.
    • Produits non créés : S'assurer que «Créer les produits manquants» est activé dans la config Shopify.

    Importer manuellement les commandes manquantes

    Si les commandes ne s'importent pas automatiquement, utiliser la fonction d'import manuel :

    1. Go to Integrations → Shopify
    2. Click Import Missing Commandes de vente
    3. Définir la plage de dates (ex. : «1er Jan au Aujourd'hui»)
    4. Click Import
    5. Vérifier le journal de connexion pour toute erreur
    ✓
    Conseil : Après avoir corrigé les taux de TVA ou les problèmes de SKU, testez toujours avec une commande Shopify récente pour confirmer que la sync fonctionne avant de traiter des centaines de commandes historiques.

    Commandes d'achat

    ▼

    Les bons de commande (BC) nous permettent de commander des stocks auprès de nos fournisseurs. Une bonne gestion des BC garantit des arrivées de stock dans les délais et des niveaux de stocks précis.

    Cycle de vie du statut d'un BC

    Statuts d'un bon de commande

    Chaque BC passe par ces étapes :

    StatutSignification
    Mis en attenteBrouillon — peut être modifié librement. Aucun engagement envers le fournisseur.
    PlacéEnvoyé/confirmé — le fournisseur s'attend à une livraison. Impossible de modifier les lignes.
    Statuts personnalisésExtensions de Placé (ex. : «Confirmé», «En transit») pour le suivi.
    RéceptionnéStock reçu en inventaire. Vous pouvez encore ajouter des lignes de coût.
    TerminéEntièrement clôturé — aucune modification possible.
    SuppriméAnnulé/retiré du système.

    Création d'un bon de commande

    Création d'un BC étape par étape

    1. Aller dans Achats → Ajouter un achat (ne jamais créer depuis une commande de vente !)
    2. Sélectionner le fournisseur (vérifier qu'il existe d'abord — ne pas créer de doublons)
    3. Définir l'entrepôt sur DEPOT_FEEL pour tout nouveau stock
    4. Ajouter des produits en utilisant le code FMSK_ pour une recherche rapide
    5. Saisir les quantités et prix unitaires
    6. Vérifier le taux de TVA : les produits français utilisent 20 % de TVA, certains produits de soin utilisent le taux réduit de 5,5 %
    7. Ajouter tout coût supplémentaire (fret, droits de douane) dans la section Lignes de coût
    8. Répartition des coûts : Toujours utiliser «Par quantité de base» (pas «Par valeur»)
    9. Vérifier le total et Enregistrer → Statut = Mis en attente
    10. Quand prêt à envoyer au fournisseur : changer le statut en Placé
    Créer un bon de commande

    Consulter les bons de commande

    Trouver et filtrer les BC

    Pour consulter les bons de commande existants :

    1. Go to Purchases → View Purchases (or use the PO Kanban view)
    2. Filtrer par : Statut, Entrepôt, Fournisseur, Plage de dates
    3. Utiliser le champ Référence fournisseur pour suivre les numéros de facture fournisseur
    Liste des bons de commande avec filtres

    Réceptionner un bon de commande

    Processus complet de réception

    Quand les marchandises arrivent d'un fournisseur, réceptionner le BC pour ajouter le stock à l'inventaire :

    1. Open the PO and click Receipt
    2. Saisir la référence fournisseur (numéro de facture) — exporté vers Odoo pour la réconciliation
    3. Saisir la date de facture fournisseur
    4. Confirmer la quantité reçue par ligne (vérifier avec le bon de livraison)
    5. Choisir : Confirmer la réception (livraison complète) ou Réception fractionnée (livraison partielle)

    Après réception : le Coût Moyen de Revient (CMR) se recalcule automatiquement, le Dernier Coût se met à jour, et le BC est exporté vers Odoo.

    ⚠
    Important : Vous ne pouvez pas définir une date de réception historique — les produits deviennent Stock disponible à partir de la date de traitement de la réception. Réceptionner rapidement à l'arrivée des marchandises.

    Réception fractionnée (livraisons partielles)

    Quand seulement une partie de la commande arrive

    Si un fournisseur n'expédie qu'une partie de votre commande, utiliser la Réception fractionnée :

    1. Open the PO and click Receipt
    2. Saisir les quantités réellement reçues (inférieures aux quantités commandées)
    3. Click Split Receipt

    Ce qui se passe :

    • Les quantités réceptionnées passent immédiatement en Stock disponible
    • Les quantités restantes créent un nouveau BC avec un suffixe (ex. : BC-00001936 → BC1-00001936/1)
    • Vous pouvez encore ajouter des coûts à la portion réceptionnée avant de compléter

    Délais fournisseurs sur les BC

    Date de livraison prévue

    Quand vous créez un BC, Unleashed calcule automatiquement la date de livraison prévue :

    • Si le fournisseur a un délai défini (ex. : 10 jours), la date de livraison est : Date de commande + 10 jours
    • Si aucun délai n'est défini, la date de livraison correspond à la date de commande (à remplir manuellement)

    Cela vous aide à planifier la disponibilité des stocks et prévoir l'arrivée des BC.

    Importer les commandes manquantes des fournisseurs

    Remplissage rétroactif des BC historiques

    Si vous devez importer des commandes passées (ex. : commandes Shopify ayant déclenché des BC dans Unleashed) :

    1. Aller dans Intégrations (si applicable) ou créer manuellement des BC
    2. Use Import/Export → Commandes d'achat for bulk import
    3. Créer un CSV avec : Code fournisseur, Code produit, Quantité, Prix unitaire, Entrepôt
    4. Télécharger et confirmer
    🔴
    CRITIQUE : Avertissement bug retour de commande
    NE PAS créer un bon de commande depuis une commande de vente. Il existe un bug connu : si vous cliquez «Créer BC» sur une commande de vente, cela crée un BC lié qui peut doubler les stocks (une fois dans la CV, une fois dans le BC), gonflant l'inventaire. Toujours créer les BC depuis Achats → Ajouter un achat.
    🔴
    CRITIQUE : Ne jamais réceptionner dans le mauvais entrepôt
    Une fois réceptionné, l'entrepôt ne peut pas être modifié. Toujours vérifier que l'entrepôt est DEPOT_FEEL avant de confirmer la réception.
    ⚠
    Important : Ne pas compléter un BC tant que tous les coûts n'ont pas été ajoutés. Vous ne pouvez pas ajouter de fret, droits de douane ou autres coûts après la complétion.
    ✓
    Conseil : Toujours saisir la référence fournisseur (numéro de facture) avant de réceptionner — indispensable pour la réconciliation Odoo en aval.

    Ajustements des stocks

    ▼

    Unleashed dispose de deux outils pour corriger les stocks : Comptage de stock et Ajustement de stock. Comprendre la différence est essentiel.

    Liste des ajustements de stock

    Comptage de stock vs Ajustement de stock

    Comptage de stock (remise à zéro complète)

    Utiliser le Comptage de stock quand vous devez remettre à zéro et ressaisir tout le stock pour un entrepôt — par exemple, lors de l'ouverture d'un nouvel emplacement ou d'un inventaire physique complet :

    1. Go to Inventory → Stock Count → Add Count
    2. Nommer le comptage (ex. : «Stocks Mars 2026»)
    3. Select the warehouse
    4. Ajouter des produits manuellement, via scan de code-barres, ou via Importation (télécharger un modèle Excel)
    5. Saisir les quantités réelles pour chaque produit
    6. Enregistrer et compléter — le système remplace les anciennes quantités par vos nouveaux comptages
    ✓
    Conseil : Pour obtenir un modèle pré-rempli, aller dans Rapports → Stock disponible, filtrer par entrepôt et exporter en XLS. Cela vous donne tous les codes produits avec les quantités actuelles — modifier la colonne quantités et réimporter dans Comptage de stock.

    Ajustement de stock (corrections individuelles)

    Utiliser l'Ajustement de stock quand vous devez corriger des produits spécifiques sans tout remettre à zéro — par exemple, articles expirés, marchandises endommagées ou retours de boutiques :

    1. Go to Inventory → Stock Adjustments → Add Ajustement
    2. Select the Entrepôt (very important — choose the right one first!)
    3. Add the product(s) you need to adjust
    4. Enter the quantity: positive = add stock, negative = remove stock
    5. Sélectionner une raison et ajouter des notes
    6. Save

    Exemple : 5 crèmes expirées à DEPOT_FEEL → sélectionner DEPOT_FEEL, trouver le produit, saisir -5, raison : «Dommage».

    2. Add adjustment lines:
      • Select the Product
      • Enter the Quantity (positive = add stock, negative = remove stock)
      • Sélectionner une raison (voir Raisons courantes d'ajustement ci-dessous)

    Raisons courantes d'ajustement

    • Comptage de stock : Écart constaté lors d'un comptage physique
    • Dommage : Produits endommagés découverts (flacons cassés, expirés, etc.)
    • Démarque : Perte inexpliquée (vol, évaporation, etc.)
    • Transfert de stock : Transfert manuel entre entrepôts (préférer utiliser Transferts entre entrepôts)
    • Échantillon : Prélevé pour démonstration ou test du personnel
    • Retour : Retour client non enregistré via le système de CV
    • Recomptage : Correction après qu'un recomptage a confirmé l'écart
    Ajustements de stock — statut complété

    Ajustements de stocks de fin de mois

    Gérer les manques de stock en fin de mois

    Lors de la clôture d'un mois dans Unleashed, vous pouvez constater que les niveaux de stock ne correspondent pas aux commandes. C'est fréquent avec les commandes PDV et les magasins physiques. Solution :

    1. Essayer de compléter en masse toutes les CV du mois
    2. Si le système affiche «stock insuffisant», noter quels produits sont en manque
    3. Créer un ajustement de stock avec la raison «Comptage de stock» pour chaque manque
    4. Relancer la complétion en masse

    Exemple : «Le canal API a vendu 5 unités de SERUM-001 mais Unleashed n'en avait que 3 en stock. Créer un ajustement : +2 unités, raison : Comptage de stock»

    ⚠
    Avertissement : Ajustements de stock directly affect inventory levels and reports. Always record a clear reason for traceability. If you adjust stock incorrectly, it skews reports and COGS calculations.

    Clôture de fin de mois

    ▼

    La clôture de fin de mois est un processus critique. Elle finalise toutes les transactions, assure la précision des stocks et clôture la période comptable pour que les rapports reflètent les chiffres définitifs.

    Processus de clôture de fin de mois étape par étape

    Étape 1 : Compléter toutes les commandes de vente ouvertes (en masse)

    Dans les derniers jours du mois, compléter toutes les CV :

    1. Menu → Commandes de vente
    2. Filter by Status: Open or Placed
    3. Sélectionner TOUTES les commandes (utiliser la case à cocher en haut pour tout sélectionner sur la page, répéter pour toutes les pages)
    4. Click Bulk Actions → Complete
    5. Click Process

    Résultat attendu : Le système traite toutes les commandes et met à jour les stocks. Certaines peuvent réussir, d'autres échouer si le stock est insuffisant.

    Étape 2 : Traiter les commandes échouées (ajustements de stocks)

    Pour les commandes n'ayant pas pu se compléter par manque de stock :

    1. Identifier quels produits sont en manque (le système les listera)
    2. Pour chaque manque, créer un ajustement de stock :
      • Go to Stock → Ajustements des stocks
      • Créer un ajustement avec Raison : Comptage de stock
      • Ajouter les unités manquantes (ex. : si SERUM-001 nécessite 3 unités de plus, ajuster +3)
      • Save
    3. Relancer la complétion en masse pour les commandes échouées

    Répéter jusqu'à ce que toutes les commandes soient complétées avec succès.

    Étape 3 : Vérifier la réconciliation des stocks

    Après avoir complété toutes les commandes, vérifier que le stock physique correspond à Unleashed :

    1. Run Stock on Hand Inquiry for each warehouse
    2. Comparer les chiffres Unleashed à votre comptage physique (si disponible)
    3. Si des écarts existent, créer des ajustements avec la raison «Recomptage» pour les corriger

    Étape 4 : Clôturer la période (comptabilité)

    Une fois toutes les commandes complétées et les stocks réconciliés, clôturer la période :

    1. Menu → Settings → Period Management (or similar, depending on your Unleashed version)
    2. Sélectionner le mois/période à clôturer (ex. : «Janvier 2026»)
    3. Click Close Period
    4. Confirmer que vous souhaitez finaliser cette période (les transactions ne peuvent généralement plus être modifiées après la clôture)

    Étape 5 : Générer les rapports finaux

    Une fois la période clôturée, lancer les rapports de fin de mois pour la comptabilité/finance :

    • Rapport Marge produit – Marge brute par produit et canal
    • Rapport Analyse des coûts – CMV total et valeur des stocks
    • Rapport Stock disponible – Niveaux de stock finaux par entrepôt
    • Exporter en CSV et fournir à votre équipe finance/comptabilité

    Problèmes courants de fin de mois & solutions

    Problème : Erreurs «Stock insuffisant» lors de la complétion en masse des commandes

    Cause : Les commandes PDV et en magasin physique ont un suivi de stock imprécis. Le système affiche moins d'unités en stock que vendues réellement.

    Solution : Créer des ajustements de stock pour corriger la différence et relancer la complétion en masse.

    Problème : Le CMV semble gonflé

    Cause : La méthode de répartition des coûts est définie sur «par valeur» au lieu de «par quantité de base».

    Solution : Vérifier la méthode de répartition des coûts de chaque produit. Changer en «par quantité de base» et recalculer.

    Problème : Les écarts de stock persistent

    Cause : Commandes manquantes, importations PDV incorrectes ou transferts de stock en double.

    Solution : Utiliser Historique des transactions pour tracer les mouvements de stock et identifier la source de l'écart.

    ⚠
    Bonne pratique : Prévoir 2-3 heures pour la clôture de fin de mois, surtout les premiers mois. Le processus deviendra plus rapide une fois que vous maîtriserez vos données et que la précision des stocks s'améliorera.
    ✓
    Conseil : Après la clôture d'une période, créer une sauvegarde ou un instantané des données. Cela protège contre les modifications accidentelles et fournit une piste d'audit.

    Règles critiques

    ▼

    Ces règles doivent toujours être respectées pour éviter des erreurs coûteuses et maintenir l'intégrité des données.

    🔴
    Règle 1 : Répartition des coûts = «Par quantité de base» (PAS «Par valeur»)
    Chaque produit DOIT avoir la répartition des coûts définie sur «par quantité de base». Utiliser «par valeur» gonfle le CMV et fausse l'analyse des marges. Vérifier ce paramètre régulièrement, surtout pour les produits importés.
    🔴
    Règle 2 : NE JAMAIS supprimer le taux de TVA 0 %
    Conserver le taux de TVA 0 % dans Paramètres → Taxes. Il est indispensable pour les commandes PDV/gros qui ne portent pas de taxe. Ajouter 20 % et 5,5 % pour Shopify est acceptable, mais NE JAMAIS supprimer le défaut 0 %.
    🔴
    Règle 3 : Ne jamais créer un BC depuis une commande de vente (bug retour de commande)
    NE PAS cliquer «Créer BC» dans une commande de vente. Cela crée un BC lié qui double les comptages de stocks. Créer les BC séparément dans le menu BC. Si vous le faites accidentellement, utiliser l'Historique des transactions pour supprimer le doublon.
    🔴
    Règle 4 : Vérifier le stock avant de vendre
    Avant de créer une CV (ou d'accepter une commande Shopify), vérifier que le stock est disponible dans le bon entrepôt. Un stock indisponible crée des retards et complique le traitement.
    🔴
    Règle 5 : Modifier les SKU produits en lot, pas un par un
    Pour synchroniser les SKU Shopify dans Unleashed, utiliser la fonction Modification en lot (Produits → Changer les codes → Lot). C'est plus rapide et évite les erreurs individuelles.
    🔴
    Règle 6 : Enregistrer les raisons de tous les ajustements
    Chaque ajustement de stock (Dommage, Démarque, Comptage de stock, etc.) doit avoir une raison documentée. Cela assure la traçabilité et la conformité aux audits.
    🔴
    Règle 7 : Confirmer les réceptions rapidement
    Dès que le stock fournisseur est reçu, cliquer sur «Recevoir» dans le BC. Cela met à jour les niveaux de stock en temps réel et prévient les écarts d'inventaire.
    🔴
    Règle 8 : Utiliser les codes d'entrepôt corrects
    Toujours utiliser le bon entrepôt (DEPOT_FEEL, CLICHY, GAITE, CHATELET, etc.). Une mauvaise sélection crée des écarts et compromet le reporting.
    🔴
    Règle 9 : Ne jamais supprimer une commande
    Si une commande est incorrecte, la marquer comme «Annulée» ou créer une note. Ne jamais la supprimer, car cela perd l'historique et rend l'audit impossible.
    🔴
    Règle 10 : Vérifier les prix régulièrement
    Les prix d'achat et de vente doivent être tenus à jour. Des prix obsolètes entraînent des marges incorrectes et des BC inexacts.
    🔴
    Règle 11 : Synchroniser les taux de TVA Shopify avant d'importer les commandes
    Créer les taux de TVA 20 % et 5,5 % dans Paramètres → Taxes AVANT d'importer les commandes Shopify. Sans taux de TVA correspondants, les commandes échouent à l'importation.
    🔴
    Règle 12 : Ne jamais valider sans confirmer d'abord
    En apprenant le système, pratiquez sans cliquer sur Compléter ou Valider. Il vaut toujours mieux ne pas valider que de valider et découvrir que vous ne pouvez pas annuler. En cas de doute, demandez à votre responsable avant de confirmer toute transaction — BC, ajustements de stock, transferts et commandes de vente. Vous pouvez enregistrer en Mis en attente/Brouillon et revenir plus tard.
    🔴
    Règle 13 : En cas de doute, contacter un responsable
    Si quelque chose semble anormal (stock négatif, prix incohérent, transfert bloqué, erreurs de synchronisation), ne pas essayer de le corriger seul. Le signaler immédiatement à votre responsable ou à l'équipe d'implémentation Unleashed.

    Tableau de résolution des problèmes

    Problème Cause Solution
    Commandes Shopify non synchronisées Taux de TVA manquants ou incorrects dans Paramètres Aller dans Paramètres → Taxes. Ajouter les taux 20 % et 5,5 %. S'assurer que le défaut 0 % n'est pas supprimé. Réimporter les commandes.
    Le CMV semble gonflé / Les marges sont incorrectes Méthode de répartition des coûts définie sur «par valeur» au lieu de «par quantité de base» Vérifier chaque produit : Produits → [Produit] → Méthode de répartition des coûts. Changer en «par quantité de base». Recalculer.
    Comptage de stock doublé / En souffrance avec inventaire doublé BC créé depuis une commande de vente (bug retour de commande) Utiliser l'Historique des transactions pour trouver le mouvement de stock en double. Le supprimer ou l'annuler. Ne jamais créer des BC depuis une CV.
    Stock négatif (-1, -5, etc.) Commandes complétées sans stock suffisant Créer un ajustement de stock avec raison «Comptage de stock» pour corriger. Vérifier l'historique des mouvements pour comprendre la cause.
    Commande bloquée / Impossible d'enregistrer Champs obligatoires manquants ou erreur de validation des données Vérifier que tous les champs obligatoires sont remplis. Vérifier les fautes de frappe dans les codes client/produit. Contacter le support si le problème persiste.
    Transfert non reçu / Bloqué en transit Transfert d'entrepôt non finalisé Aller dans Transferts entre entrepôts, trouver le transfert, cliquer «Recevoir» pour finaliser. Vérifier que les quantités correspondent.
    Le stock diffère du comptage physique Ventes non enregistrées, dommages, vol ou erreur de comptage Créer un ajustement de stock avec raison «Comptage de stock» pour enregistrer la différence. Enquêter sur la cause séparément.
    Fournisseur introuvable Fournisseur non créé dans Unleashed Vérifier l'orthographe. Si absent, créer un nouveau fournisseur : Menu → Fournisseurs → + Nouveau fournisseur. Définir l'entrepôt par défaut et le délai.
    Client introuvable La fiche client n'existe pas Créer le client : Menu → Clients → + Nouveau client. Utiliser des codes simples comme «SHOPIFY», «GAITE-PDV», «ACME-WHOLESALE».
    Incompatibilité SKU Shopify (produits ne correspondant pas) Le SKU Shopify ne correspond pas au code produit Unleashed Utiliser Produits → Changer les codes → Lot pour synchroniser les SKU Shopify. Télécharger le modèle, importer depuis l'export Shopify, mise à jour en masse.

    Erreurs courantes et comment les corriger

    ▼

    Les erreurs arrivent — ce qui compte, c'est de savoir comment les corriger. Les transactions complétées dans Unleashed ne peuvent pas être supprimées ou modifiées, vous devez donc utiliser des transactions correctives à la place.

    Réceptionné le mauvais produit ou la mauvaise quantité

    1. Créer un retour fournisseur pour annuler la réception incorrecte
    2. Le stock sera déduit de l'inventaire
    3. Créer un bon de commande corrigé avec les bons produits/quantités

    Réceptionné dans le mauvais entrepôt

    Vous ne pouvez pas changer l'entrepôt après la réception. À la place :

    1. Créer un transfert d'entrepôt depuis le mauvais entrepôt vers le bon
    2. Traiter et confirmer le transfert

    Complété un BC avec des erreurs

    Une fois un BC complété, il ne peut pas être rouvert. À la place :

    1. Créer un retour fournisseur pour annuler la transaction
    2. Créer un nouveau bon de commande corrigé

    Écart de stock (physique vs. système)

    Si votre comptage physique ne correspond pas à ce qu'Unleashed affiche :

    1. Effectuer un comptage de stock pour réconcilier formellement
    2. Créer des ajustements de stocks pour corriger l'écart
    3. Toujours enregistrer une raison (Comptage de stock, Dommage, Démarque, etc.)

    BC créé accidentellement depuis une commande de vente (bug retour de commande)

    1. Go to Reports → Transaction Inquiry
    2. Trouver le mouvement de stock en double créé par le BC lié
    3. Supprimer ou annuler l'entrée en double
    4. Créer un BC correct depuis Achats → Ajouter un achat si encore nécessaire
    🔴
    À retenir : Les transactions complétées dans Unleashed NE PEUVENT PAS être supprimées ou modifiées. Vous devez toujours utiliser des transactions correctives (Retours fournisseurs, Ajustements des stocks, Transferts entre entrepôts) pour corriger les erreurs.
